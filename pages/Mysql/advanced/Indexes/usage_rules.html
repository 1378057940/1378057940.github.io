最左前缀法则(联合索引)

  1. 正常使用索引
#使用的索引长度为61
explain select * from emp where work_no = '2' and `age` = 10 and name = '张无忌';
explain select * from emp where `age` = 10 and `name` = '张无忌' and work_no = '2';
explain select * from emp where work_no = '2' and `age` = 10 and gender = '男' and name = '张无忌';
#使用的索引长度为56
explain select * from emp where work_no = '2' and `age` = 10;
#使用的索引长度为13
explain select * from emp where work_no = '2';
#使用的索引长度为13,因为跳过了`age`字段,所以`age`字段之后的索引全部失效
explain select * from emp where work_no = '2' and name = '张无忌';
  1. 未使用索引
explain select * from emp where `age` = 10 and `name` = '张无忌';
explain select * from emp where age = 10;
explain select * from emp where `name` = '张无忌';

TIP

  1. 指的是查询从索引的最左列开始,并且不跳过索引中的列
  2. 如果跳过某一列,则跳过的一列之后的所有列索引全部失效
  3. 书写顺序不影响索引的使用,只要对应的索引字段出现就可以,不可跳过字段出现
#两条sql是等价的
explain select * from emp where work_no = '2' and `name` = '张无忌' and age = 10;
explain select * from emp where `name` = '张无忌' and age = 10 and work_no = '2';

范围查询

#索引长度15,`age`使用了范围查询(<>),所以`age`之后所有的索引列索引全部失效
explain select * from emp where work_no = '2' and age > 10 and `name` = '张无忌';
explain select * from emp where work_no = '2' and age < 10 and `name` = '张无忌';
#索引长度58,使用`>=`或`<=`索引正常使用
explain select * from emp where work_no = '2' and age >= 10 and `name` = '张无忌';
explain select * from emp where work_no = '2' and age <= 10 and `name` = '张无忌';

索引列运算

explain select * from emp where substring(work_no,10,2) = 15;

WARNING

不要在索引列上进行运算操作,否则索引会失效

字符串不加引号

#`work_no`没加单引号,后续所有索引失效
explain select * from emp where work_no = 2 and age <= 10 and `name` = '张无忌';
#数字类型的索引列可以加单引号,也可以不加
explain select * from emp where work_no = 2 and age <= '10' and `name` = '张无忌';

模糊查询

#未使用索引
explain select * from emp where work_no like '%a' and age <= 10 and `name` = '张无忌';
explain select * from emp where work_no like '_a' and age <= 10 and `name` = '张无忌';
explain select * from emp where work_no like '_a_' and age <= 10 and `name` = '张无忌';
explain select * from emp where work_no like '%a%' and age <= 10 and `name` = '张无忌';
#正常使用索引
explain select * from emp where work_no like 'a_' and age <= 10 and `name` = '张无忌';
explain select * from emp where work_no like 'a%' and age <= 10 and `name` = '张无忌';

使用数字模糊查询也无法使用索引

explain select * from emp where work_no like '1%' and age <= 10 and `name` = '张无忌';

WARNING

尾部模糊可以正常使用索引
头部模糊或者数字模糊无法使用索引

or连接

#`dept_id`没有建立索引,涉及的索引失效
explain select * from emp where work_no = '3' or dept_id = 5;
#使用了主键索引和work_no联合索引
explain select * from emp where id = 3 or dept_id = 5 and work_no = '10';
#只使用了work_no联合索引
explain select * from emp where (id = 3 or dept_id = 5) and work_no = '10';

TIP

用or分割开的条件,如果or前的条件中的列有索引,而后面的列中没有索引,那么所涉及的索引都不会生效

数据分布影响(使用索引比全表扫描慢,则放弃索引)

#使用索引
explain select * from emp where age >= 70;
#未使用索引
explain select * from emp where age >= 50

TIP

Mysql评估大部分age都是大于50的,所以放弃索引,使用全表扫描
只有少部分age大于70,所以使用索引 说白了少数走索引,多数走全表

sql提示

#可能使用到的索引`idx_emp_work_age_name`和`idx_emp_work`,实际使用的索引为`idx_emp_work_age_name`
explain SELECT * FROM `emp` where work_no = '10';
#建议Mysql使用指定的索引(只是建议使用,Mysql如果觉得指定的索引速度不快,则不会使用)
explain SELECT * FROM `emp` use index(idx_emp_work) where work_no = '10';
#忽略`idx_emp_work`索引
explain SELECT * FROM `emp` ignore index(idx_emp_work) where work_no = '10';
#强制使用`idx_emp_work`索引
explain SELECT * FROM `emp` force index(idx_emp_work) where work_no = '10';

前缀索引

  1. 语法
create index 索引名 表名 (字段名(长度));
  1. 前缀长度

当字段类型为字符串(varchar,text等)时,有时候需要索引很长的字符串,这会让索引变得很大,查询时,浪费大量的磁盘IO,影响查询效率
此时可以只将字符串的一部分前缀建立索引,提高索引效率

#该字段的选择性
select count(distinct 字段名) / count(*) from 表;
#该字段前5个字符的选择性
select count(distinct substring(字段名,1,5)) / count(*) from 表;

TIP

可以根据索引的选择性来决定,而选择性是指不重复的索引值(基数)和数据表记录的总数比值,索引选择性越高则查询效率越高
唯一索引的选择性为1,是最好的索引选择性
